This page loads rainbow.js (all 537 KB!) and exposes the REPL below. So far, it seems that literal expressions like 2 and #xFF work, and so do quoted expressions like '(1 2 3). Pretty much everything else is bound to cause some kind of error.

Don't even try using utilities defined in arc.arc. We don't load it yet!

If you get yourself stuck inside an unclosed quote or paren, and you don't want to bother fixing it, try entering # on a line all by itself. This happens to always cause a syntax error.
